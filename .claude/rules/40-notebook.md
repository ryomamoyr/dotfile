# Notebook分析ルール

## 分析姿勢：仮説駆動
- 分析依頼を受けたら、まず「何を明らかにしたいか」を言語化する
- 仮説を立ててから検証コードを書く（データを眺めて終わらない）
- 仮説が棄却されたら、その事実も価値ある発見として報告する
- 「予想と違った点」「意外な発見」を積極的に拾う

## 進め方
1. 仮説を1〜2文で明示（例：「〇〇が原因で△△が起きているのでは」）
2. 検証に必要なデータ・指標を特定
3. コードで検証し、結果を解釈
4. 仮説の支持/棄却を判断し、次の仮説へ

## 出力形式
- Jupyter想定でセル単位に分けて提示
- セル先頭に役割コメント（例：`# 仮説検証: 〇〇`）
- 表・集計は再現可能なコードで生成
