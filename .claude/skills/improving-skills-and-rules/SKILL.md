# Skills/Rules改善

ユーザーの作業パターンから再利用可能な知識を抽出し、Skills/Rulesとして定着させる。意図しない変更はユーザーの信頼を損なうため、提案→承認のフローを守る。

## トリガー条件

以下を検出したら改善を提案する：

1. **繰り返し指示** — 同じ指示を2回以上受けた場合、Ruleに定着させる価値がある
2. **矛盾検出** — 既存ルールと異なる指示を受けた場合、どちらが正しいか確認する
3. **新パターン** — 汎用化できそうな手順が確立された場合、Skillとして切り出す
4. **明示的依頼** — 「これルール化して」「スキルに追加して」

## 提案フォーマット

```
**Skills/Rules改善提案**

**種別**: [新規作成 | 更新 | 削除]
**対象**: `.claude/[skills|rules]/xxx`
**理由**: [なぜこの変更が有効か]

**変更内容**:
```diff
- 削除される内容
+ 追加される内容
```

この変更を適用しますか？
```

## 判断基準

| 状況 | アクション |
|------|-----------|
| 2回同じ指示 | 提案を検討 |
| 3回同じ指示 | 提案する |
| 既存ルールと矛盾 | 確認して提案 |
| 「ルール化して」 | 即座に提案 |

## 更新時の確認事項

- 既存のSkill/Ruleと重複しないか
- 他のSkill/Ruleと矛盾しないか
- 適切な粒度か（1ファイル = 1関心事）
- 小さな変更を頻繁に行う（大きな変更は分割する）
